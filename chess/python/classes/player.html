Player¶

class games.chess.Player¶

Bases: games.chess.game_object.GameObject

The class representing the Player in the Chess game.

A player in this game. Every AI controls one player.

client_type¶

What type of client this is, e.g. ‘Python’, ‘JavaScript’, or some other language. For potential data mining purposes.

color¶

The color (side) of this player. Either ‘White’ or ‘Black’, with the ‘White’ player having the first move.

file_direction¶

The direction your checkers must go along the file (y) axis until they reach the other side.

game_object_name¶

String representing the top level Class that this game object is an instance of. Used for reflection to create new instances on clients, but exposed for convenience should AIs want this data.

id¶

A unique id for each instance of a GameObject or a sub class. Used for client and server communication. Should never change value after being set.

in_check¶

true if this player is currently in check, and must move out of check.

log(message)¶

adds a message to this game object’s log. Intended for debugging purposes.

Args:
message (str): A string to add to this GameObject’s log. Intended for debugging.
logs¶

Any strings logged will be stored here when this game object logs the strings. Intended for debugging.

lost¶

if the player lost the game or not

made_move¶

If the Player has made their move for the turn. true means they can no longer move a piece this turn.

name¶

The name of the player

other_player¶

this player’s opponent in the game.

pieces¶

All the unpcaptured chess pieces owned by this player.

reason_lost¶

the reason why the player lost the game

reason_won¶

the reason why the player won the game

time_remaining¶

The amount of time (in ns) remaining for this AI to send commands.

won¶

if the player won the game or not
