Job¶

class games.catastrophe.job.Job¶

Bases: games.catastrophe.game_object.GameObject

The class representing the Job in the Catastrophe game.

Information about a Unit’s job.

action_cost¶

The amount of energy this Job normally uses to perform its actions.

Return type:float
carry_limit¶

How many combined resources a Unit with this Job can hold at once.

Return type:int
game_object_name¶

String representing the top level Class that this game object is an instance of. Used for reflection to create new instances on clients, but exposed for convenience should AIs want this data.

Return type:str
id¶

A unique id for each instance of a GameObject or a sub class. Used for client and server communication. Should never change value after being set.

Return type:str
log(message)¶

Adds a message to this GameObject’s logs. Intended for your own debugging purposes, as strings stored here are saved in the gamelog.

Parameters:message (str) – A string to add to this GameObject’s log. Intended for debugging.
logs¶

Any strings logged will be stored here. Intended for debugging.

Return type:list[str]
moves¶

The number of moves this Job can make per turn.

Return type:int
regen_rate¶

The amount of energy normally regenerated when resting at a shelter.

Return type:float
title¶

The Job title.

Return type:str
upkeep¶

The amount of food per turn this Unit consumes. If there isn’t enough food for every Unit, all Units become starved and do not consume food.

Return type:int
